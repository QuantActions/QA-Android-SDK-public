# QA

[androidJvm]\
open class [QA](index.md)

This is the main element one needs to use to access all the functionality of the QA SDK. As a singleton it can be called easily from anywhere in the code and gives access to all the possible interactions with the QA server, as well as some functions to retrieve coarse statistics on the number of taps on the device. Since most of the calls are server interactions, the SDK offers the possibility to pass a runnable thread to each call in order to asynchronously deal with the server response. One has to simply extend [QARunnable](../-q-a-runnable/index.md) and create a proprietary thread to handle responses. In general a response contains a flag representing the call type and metadata associated with the response, please see the doc for the specific calls to understand the metadata include depending on the request. In general the response is a [QAJSON](../-q-a-j-s-o-n/index.md) dictionary will all the info.
